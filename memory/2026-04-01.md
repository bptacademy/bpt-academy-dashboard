# Memory — 2026-04-01

## BPT Academy APK Crash Debugging Session

### Root Causes Found (Android APK crashing on launch, silent)

1. **`xlsx` top-level import in `ReportsScreen.tsx`**
   - `xlsx` uses Node.js built-ins (`stream`, `crypto`, `zlib`) not available in React Native
   - Metro bundles all imports eagerly → crashes on startup before anything renders
   - Fix: moved `require('xlsx')` inside `handleExport()` (lazy load, only runs when tapped)

2. **Supabase env vars missing from EAS builds**
   - `supabase.ts` reads `process.env.EXPO_PUBLIC_SUPABASE_URL` etc.
   - `.env` file exists at `mobile/.env` but EAS cloud builds do NOT pick up `.env` files
   - `eas.json` had no `env` block → `createClient(undefined, undefined)` threw on startup
   - Fix: added `env` block to all profiles (development, preview, production) in `eas.json`

3. **`expo-notifications` not installed as native module**
   - Was imported at top level of `notifications.ts` and `useNotifications.ts`
   - No `expo-notifications` plugin in `app.json`, no `google-services.json`
   - Fix: stubbed both files as no-ops, removed notifications plugin from `app.json`
   - Notifications pending proper Firebase setup (future task)

4. **`SafeAreaProvider` missing from `App.tsx`**
   - `useSafeAreaInsets()` called in tab bar but no `SafeAreaProvider` at root
   - Caused crash in Expo Go on iPhone
   - Fix: wrapped app in `<SafeAreaProvider>` in `App.tsx`

### Key Lesson
- EAS builds do NOT read `.env` files — always use `env` block in `eas.json` for cloud builds
- Never use top-level imports of Node.js-dependent libraries (xlsx, etc.) in React Native
- Always wrap app root with `SafeAreaProvider` when using `useSafeAreaInsets()`
